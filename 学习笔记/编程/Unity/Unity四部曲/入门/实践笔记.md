## 场景切换

- 注意：
	对于要切换的目标场景
	需要将其先加载到场景列表中
	否则场景的切换无法实现
	
	如何将目标场景加载到场景列表中？
	Unity 编辑器工具栏中 `File/Build Settings` 打开导出窗口
	像这样：![[Pasted image 20260317134544.png]]
	上方的 `Scenes In Build` 即场景列表
	点击 `Add Open Scenes` 即可将现在 Hierachy 窗口中的所有场景加载到列表中
	也可以将场景文件拖拽到列表中直接添加
	场景列表中每个场景中的序号代表该场景在游戏运行时的加载顺序
	序号越小的场景越先被加载
	0 序号表示该场景在游戏运行时即被加载 一般作为游戏主界面或登录界面

## 模型资源导入

- 模型由什么构成？
	骨 (骨骼)：非必须 有动作的模型才需要
	肉 (网格面片)：必须 决定了模型的轮廓
	肉 (网格面片)：必须 决定了模型的颜色效果

- Unity支持的模型格式
	.fbx
	.dae
	.3ds
	.dxf
	.obj

- 关于Unity模型导出的官方文档
	https://docs.unity.cn/cn/2019.4/Manual/CreatingDCCAssets.html

- 导出的模型的重要规则
	Unity中的模型面朝向朝模型坐标轴的 Z 轴
	缩放大小单位要注意

# 数据持久化 PlayerPrefs


什么是数据持久化？
数据持久化就是将游戏运行时产生的数据存储到硬盘当中
即将内存的数据存储到硬盘当中进行保存
在下一次游戏运行时通过读取硬盘中的数据实现上一次游戏数据的继承
在这一过程中数据被持久保存在硬盘里
这就是数据持久化 描述数据的存取读取的两个过程
而 PlayerPrefs 就是实现数据持久化的方法之一

# GUI

## GUI 工作原理和主要作用

- 概述
	GUI 即由代码驱动的 UI 系统
	它的特点是只能在游戏运行时才能看到 UI 显示
	它一般作为程序员的调试工具 而不是玩家 UI 显示
	可以为脚本组件创建自定义检视面板
	或创建新的编辑器窗口和工具以拓展 Unity 本身

- 见图：
	![[Pasted image 20260319180252.png]]

## GUI 基本控件

- GUI 控件绘制的共同点
	绘制这些 UI 就是去
	`OnGUI()` 这个类似生命周期函数的特殊函数
	里面调用对应 UI 控件的静态函数
	
	而这些函数的参数一般包含：
	- `Rect` 描述该 UI控件 位置 xy 坐标 和 wh 尺寸
	- `string` UI 中的显示文本
	- `Texture` 显示图片信息
	- `GUIContent` 综合信息
	- `GUIStyle` 自定义样式
	
	每一种 UI控件 都有多种重载
	无非就是这些参数的排列组合
	注意：排布 UI 的位置坐标原点(0,0) 为游戏窗口的左上角

## 小项目

- **GUI的缺点**
	1. 不能所见即所得 需要运行游戏才能看到 UI 显示
	2. 无法实现分辨率自适应
	3. 无法很好的控制控件的绘制顺序 层级显示关系

- **目的**
	解决GUI的两个缺点 将这些基础 UI 控件做成预制体以防止代码的重复书写

### 控件位置信息计算类

- 作用：
	实现了两个九宫格计算偏移的方法
	1. 屏幕九宫格
	2. 控件中心位置九宫格
	有允许自行修改当前坐标系下的偏移的 Vector2 成员变量 `pos`
	只需要调用属性 `Pos` 就可以得到控件在该规则下的坐标
	解决了原本 GUI 无法实现屏幕自适应的问题

### 这是画布吗？

~~这一定是画布吧~~

- 作用：
	`CustomGUIRoot` 脚本在编辑模式下始终执行
	其内部的 `GUI()` 函数将持续绘制 GUI控件
	挂载了该脚本的物体作为根物体 ~~画布~~
	将 GUI控件 作为其子物体 使得所有的绘制统一在该 `GUI()` 函数中完成
	而作为 GUI控件 的物体自身不需要有 `GUI()` 该函数
	只需要有自己的绘制方法 `DrawGUI()`
	解决了 GUI 不能所见即所得的问题 在编辑模式下也能绘制 GUI
	解决了无法自由控制 GUI控件 绘制显示层级的问题
